           Integrazioni
          
MCP Server — client LLM esterni

          

            SentinelNet è utilizzabile da client LLM esterni (Claude Desktop, LM Studio, Cline…) tramite il protocollo
            MCP (Model Context Protocol): il ponte mcp_server.py gira sulla macchina del client,
            si autentica con un account SentinelNet e inoltra le richieste all'API REST. Ruoli, tenant e blacklist comandi
            restano applicati lato server. Consiglio: crea un account dedicato (viewer per sola lettura,
            operator se servono i comandi CLI).
          

        

      


      

        

          
 Configurazione client

          
        

        

          Snippet per claude_desktop_config.json (o equivalente del tuo client). Sostituisci utente e password
          con l'account dedicato; il percorso deve puntare alla copia locale di mcp_server.py.
        

        

      


      

        
 Tool esposti

        

          Scegli quali tool i client MCP possono usare. I tool disattivati spariscono dall'elenco del client e ogni
          chiamata viene rifiutata. La modifica vale per tutti i client collegati (ricaricano l'elenco entro ~1 minuto).
        

        

        

          
          
        

      


      

        
 MCP Client (anteprima)

        

          Abilita la tab sperimentale "MCP Client": SentinelNet come client verso server MCP esterni (Jira, ServiceNow…) via Streamable HTTP. Funzionalita' in anteprima.
        

        
        
      


      

        
 FortiGate LIVE (anteprima)

        

          Abilita la tab sperimentale "FortiGate LIVE": gestione token API e visualizzazione degli oggetti firewall (indirizzi, policy, servizi) in tempo reale. Funzionalita' in anteprima.

           Amministrazione
          
Impostazioni

          

            Configurazione dell'istanza, raggruppata per ambito: esposizione in rete, sicurezza dei comandi, telemetria e parametri applicativi. Le impostazioni contrassegnate richiedono il riavvio dell'applicazione; le variabili d'ambiente, se impostate, hanno sempre la precedenza.
          

        

      


      
      

        
 Esposizione in rete

        

          Scegli su quale indirizzo SentinelNet resta in ascolto. 0.0.0.0 espone l'app su tutte le interfacce (raggiungibile dalla LAN); 127.0.0.1 la limita a questo PC.
        

        

          

        

      


      
      

        
 Blacklist comandi CLI

        

          Gli amministratori bypassano sempre la blacklist dei comandi distruttivi (i comandi restano tracciati nell'audit log). Qui decidi se applicarla agli operatori.
        

        
        

      


      
      

        
 Observability (IPFIX/sFlow/syslog/NetFlow)

        

          Configura i listener di telemetria di rete. Le modifiche richiedono il riavvio dell'applicazione per avere effetto.
